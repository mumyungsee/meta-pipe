# Step 1: 도메인 용어 & 요구사항 정의

> **파이프라인**: 카드뉴스 자동화 도구
> **목적**: 카드뉴스 도메인의 용어를 코드베이스 전체에서 일관되게 사용하고, 핵심 기능 범위를 확정한다.

---

## 1. 도메인 용어 사전

### 1.1 구조 용어

| 한국어 | 영어 | 코드명 | 설명 |
| --- | --- | --- | --- |
| 프로젝트 | Project | `Project` | 카드뉴스 한 세트의 최상위 컨테이너 |
| 슬라이드 | Slide | `Slide` | 개별 카드 1장 (= 페이지). "카드"와 혼용하지 않음 |
| 표지 | Cover | `cover` | 첫 번째 슬라이드. 메인 타이틀 + 흥미 유발 |
| 본문 | Body | `body` | 2~N장. 핵심 메시지 전달 |
| 마지막 장 | Closing | `closing` | CTA, 브랜드 로고, 출처 표기 |
| 요소 | Element | `Element` | 슬라이드 위에 배치되는 개별 객체 (텍스트, 이미지, 도형 등) |

### 1.2 디자인 용어

| 한국어 | 영어 | 코드명 | 설명 |
| --- | --- | --- | --- |
| 톤앤매너 | Tone & Manner | `StyleGuide` | 전체 카드뉴스의 시각적 분위기 통일 기준 |
| 레이아웃 | Layout | `layout` | 요소들의 배치 구조. 슬롯 시스템으로 추상화 |
| 행간 | Line Height | `line_height` | 줄 간격. 폰트크기 x 1.618 (황금비) 권장 |
| 여백 | Margin/Padding | `margin`/`padding` | 가독성의 핵심. 충분히 확보 |
| 판형 | Aspect Ratio | `aspect_ratio` | 이미지 비율 (1:1, 4:5, 16:9, 9:16) |
| 템플릿 | Template | `Template` | 재사용 가능한 레이아웃 + 스타일 조합 |

### 1.3 콘텐츠 용어

| 한국어 | 영어 | 코드명 | 설명 |
| --- | --- | --- | --- |
| 카피 | Copy | `copy` | 슬라이드의 텍스트 콘텐츠 |
| CTA | Call to Action | `cta` | 행동 유도 문구 (마지막 장에 배치) |
| 스토리라인 | Storyline | `storyline` | 슬라이드 간 메시지 흐름 |

### 1.4 기술 용어

| 한국어 | 영어 | 코드명 | 설명 |
| --- | --- | --- | --- |
| 렌더링 | Rendering | - | 데이터 모델 → 이미지 변환 과정 |
| 내보내기 | Export | `ExportConfig` | 생성된 이미지를 파일로 출력 |
| 플랫폼 어댑터 | Platform Adapter | `adapter` | 플랫폼별 사이즈/포맷 변환 모듈 |

## 2. 지원 플랫폼 & 판형 매트릭스

| 플랫폼 | 판형 | 비율 | 해상도 (px) | 최대 용량 |
| --- | --- | --- | --- | --- |
| 인스타그램 피드 | 정방형 | 1:1 | 1080 x 1080 | 30MB |
| 인스타그램 피드 | 세로 | 4:5 | 1080 x 1350 | 30MB |
| 인스타그램 스토리 | 세로 전체 | 9:16 | 1080 x 1920 | 30MB |
| 페이스북 | 가로 | 16:9 | 1920 x 1080 | 10MB |
| 블로그 (네이버/티스토리) | 가로 | 16:9 | 1920 x 1080 | - |
| 카카오톡 채널 | 정방형 | 1:1 | 800 x 800 | 5MB |

**기본값**: 인스타그램 1:1 (1080x1080) - 가장 범용적

## 3. 사용자 페르소나

| 페르소나 | 설명 | 핵심 니즈 |
| --- | --- | --- |
| **마케터** | 기업/브랜드 SNS 담당자 | 대량 생산, 브랜드 일관성, 빠른 제작 |
| **콘텐츠 크리에이터** | 개인 블로거, 유튜버 | 쉬운 사용, 다양한 스타일, 무료 리소스 |
| **기업 홍보팀** | 공공기관/기업 홍보 담당 | 접근성 준수, 법적 안전, 공식적 톤 |

## 4. MVP 기능 범위

### 4.1 MVP (v1.0)

| 기능 | 설명 |
| --- | --- |
| 텍스트 → 슬라이드 분할 | 입력 텍스트를 카드뉴스 슬라이드로 자동 구성 |
| 기본 템플릿 시스템 | cover/body/closing 각 2개 이상 레이아웃 |
| HTML/CSS 렌더링 | Puppeteer 기반 이미지 생성 |
| 3개 플랫폼 내보내기 | 인스타그램 1:1, 페이스북 16:9, 스토리 9:16 |
| 오픈소스 폰트/이미지 | 나눔스퀘어, 프리텐다드 + Pexels/Unsplash |
| 라이선스 추적 | 사용된 이미지/폰트의 라이선스 정보 기록 |
| alt text 자동 생성 | 각 슬라이드의 접근성 텍스트 |

### 4.2 Future (v2.0+)

| 기능 | 설명 |
| --- | --- |
| AI 이미지 생성 | DALL-E/Midjourney 연동 |
| 플랫폼 직접 게시 | Instagram Graph API, Facebook API |
| 브랜드 키트 관리 | 여러 브랜드의 스타일 가이드 저장/전환 |
| 협업 기능 | 여러 사용자 동시 편집 |
| A/B 테스트 | 여러 버전 생성 후 성과 비교 |
| 영상 변환 | 카드뉴스 → 릴스/쇼츠 자동 변환 |

---

## 주의사항

- "카드"와 "슬라이드"를 혼용하지 않음. 코드에서는 `Slide` 통일
- 플랫폼별 사이즈 제약은 설정 파일로 관리 (하드코딩 금지)
- 판형 정보는 수시로 변경될 수 있으므로 `config/platforms.json`에서 관리
