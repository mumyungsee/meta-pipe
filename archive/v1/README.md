# meta-pipe v1 Archive

> 아카이브 일시: 2026-04-01

## v1 개요

도메인 지식 없이 AI가 도메인을 조사하고, 파이프라인을 설계/실행하는 3단계 구조:
- Phase A: Discover (도메인 조사)
- Phase B: Design Pipeline (파이프라인 설계)
- Phase C: Execute (단계별 실행)

## v1에서 배운 것

- 프로그램 개발은 도구가 갖춰져 있어 바로 실행 가능하지만, 비개발 도메인(이미지, 영상 등)은 도구 세팅이 선행되어야 한다.
- 사용자 맥락(목표, 역량, 환경) 파악 없이 파이프라인을 만들면 실행 불가능한 파이프라인이 나올 수 있다.
- 파이프라인을 만든 뒤에도 PDCA 루프로 개선하는 과정이 필수다.

## v2로 전환한 이유

- 컨설팅(인터뷰) 단계 추가 필요
- Step별 실행 모드(auto/assist/manual) 태깅 필요
- 도구 세팅 과정을 파이프라인에 포함 필요
- 결과물 평가/개선 루프 필요
