# Step 8: 통합 테스트 & 품질 검증

> **파이프라인**: 카드뉴스 자동화 도구
> **목적**: 전체 파이프라인(텍스트 입력 → 이미지 출력)을 end-to-end로 검증한다.
> **approval_required**: false (테스트/검증 단계, 결과만 리포트)

---

## 1. E2E 테스트 시나리오

### 1.1 도메인별 테스트 (3개)

| 시나리오 | 입력 | 기대 결과 |
| --- | --- | --- |
| **마케팅** | "여름 세일 프로모션" (주제어) | 5~7장, 밝은 톤, CTA 포함 |
| **교육** | 블로그 글 1000자 (원문 텍스트) | 7~10장, 정보 전달형, 데이터 활용 |
| **뉴스** | 뉴스 기사 URL (URL 입력) | 5~8장, 중립 톤, 출처 명시 |

### 1.2 각 시나리오 검증 항목

```
입력 처리:
  [ ] 입력 유형 정확히 감지 (topic/text/url)
  [ ] 슬라이드 수 적절 (5~10장)
  [ ] cover → body → closing 순서 정확

콘텐츠 품질:
  [ ] 제목 15자 이내
  [ ] 본문 50자 이내
  [ ] 슬라이드당 메시지 1개
  [ ] 스토리 흐름 자연스러움 (수동 검증)

렌더링 품질:
  [ ] 1080px 기준 선명
  [ ] 한글 폰트 정상 렌더링 (자간, 행간, 장평)
  [ ] 이미지 위 텍스트 가독성 확보
  [ ] 배경과 텍스트 대비율 3:1 이상

내보내기:
  [ ] 3개 플랫폼 동시 내보내기 정상
  [ ] 파일 용량 제한 준수
  [ ] 파일 네이밍 컨벤션 준수

접근성/법적:
  [ ] 모든 슬라이드 alt_text 존재
  [ ] 라이선스 보고서 생성
  [ ] 텍스트 전용 버전 생성
```

## 2. 시각적 품질 체크리스트

### 2.1 자동 검증 가능

| 항목 | 기준 | 자동화 |
| --- | --- | --- |
| 텍스트 글자 수 | 제목 ≤15자, 본문 ≤50자 | 문자열 길이 체크 |
| 이미지 해상도 | 1080px 이상 | 이미지 메타데이터 |
| 파일 용량 | 플랫폼별 제한 이내 | 파일 크기 체크 |
| 대비율 | 텍스트/배경 3:1 이상 | 색상 대비 계산 |
| alt_text 존재 | 모든 슬라이드 | null 체크 |
| 라이선스 정보 | 모든 이미지 | LicenseInfo 존재 |
| 슬라이드 순서 | cover → body → closing | type 순서 검증 |

### 2.2 수동 검증 필요

| 항목 | 기준 | 방법 |
| --- | --- | --- |
| 가독성 | 모바일 375px에서 읽기 편한가 | 실제 모바일 미리보기 |
| 일관성 | 슬라이드 간 디자인 통일 | 목시 확인 |
| 여백 | 답답하지 않은가 | 시각 확인 |
| 폰트 크기 | 본문 25pt 이상 체감 | 시각 확인 |
| 이미지 품질 | 깨짐/흐림 없는가 | 시각 확인 |
| 한글 렌더링 | 줄바꿈, 자간, 행간 자연스러운가 | 시각 확인 |
| 스토리 흐름 | 논리적 연결 자연스러운가 | 내용 검토 |

## 3. 성능 벤치마크

| 항목 | 목표 | 측정 방법 |
| --- | --- | --- |
| 콘텐츠 생성 (LLM) | 10장 기준 30초 이내 | Stage 1~4 소요시간 합산 |
| 렌더링 (Puppeteer) | 슬라이드 1장당 3초 이내 | 렌더링 함수 실행 시간 |
| 전체 파이프라인 | 10장 카드뉴스 60초 이내 | 입력 → 전체 출력 시간 |
| 내보내기 | 3개 플랫폼 x 10장 10초 이내 | 리사이즈 + 저장 시간 |

## 4. 실제 SNS 업로드 검증

```
렌더링 결과를 실제 플랫폼에 업로드하여 확인:

인스타그램:
  [ ] 피드 업로드 시 잘림 없음
  [ ] 여러 장 업로드 시 순서 정확
  [ ] 모바일에서 텍스트 가독성

페이스북:
  [ ] 가로 이미지 정상 표시
  [ ] 링크 미리보기 시 깨짐 없음

블로그:
  [ ] 이미지 선명도 유지
  [ ] 본문에 삽입 시 레이아웃 정상
```

## 5. 품질 점수 카드

각 테스트 시나리오에 대해 품질 점수를 매긴다:

```
품질 점수 (1-5):
  5: 전문 디자이너 수준, 바로 게시 가능
  4: 약간의 수정으로 게시 가능
  3: 컨셉은 좋으나 수정 필요
  2: 기본 구조만 갖춤, 대폭 수정 필요
  1: 사용 불가

평가 항목별:
  - 콘텐츠 품질: /5
  - 디자인 품질: /5
  - 기술 품질 (렌더링, 해상도): /5
  - 접근성/법적 준수: /5
  - 종합: /5

목표: 종합 3.5/5 이상 (약간의 수정으로 게시 가능 수준)
```

## 주의사항

- 모바일 화면 시뮬레이션 필수 (375px 기준) - 카드뉴스는 대부분 모바일에서 소비
- 한글 특화 테스트: 줄바꿈 위치, 자간, 행간, 장평이 자연스러운지 반드시 확인
- 실제 SNS 업로드 테스트는 가능한 경우에만 (테스트 계정 활용)
- LLM 프롬프트 품질은 반복 테스트로 개선 [검토 필요]
