## 1. 카드뉴스 핵심 용어 표준

### 1.1 카드뉴스란

슬라이드 형태로 넘겨 보는 이미지 기반 콘텐츠. 주로 SNS(인스타그램, 페이스북, 카카오톡, 네이버 포스트)에서 정보 전달 목적으로 사용된다.

### 1.2 구성요소 용어

| 용어 | 영문 | 설명 |
| --- | --- | --- |
| **표지(커버)** | Cover Page | 첫 번째 슬라이드. 메인 타이틀 + 서브 타이틀 + 배경 이미지. 흥미 유발이 핵심 |
| **본문** | Body Pages | 2\~10장. 제목 + 내용 텍스트 + 배경 이미지/그래픽 |
| **마지막 장** | Closing Page | CTA(Call to Action), 브랜드 로고, 출처 표기 등 |
| **슬라이드/카드** | Slide/Card | 개별 페이지 단위 |
| **톤앤매너** | Tone & Manner | 전체 카드뉴스의 시각적 분위기 통일 기준 |
| **레이아웃** | Layout | 텍스트, 이미지, 그래픽 요소의 배치 구조 |
| **행간** | Line Height | 줄 간격. 폰트크기 x 1.618 (황금비) 권장 |
| **여백** | Margin/Padding | 텍스트 주변 빈 공간. 가독성의 핵심 |

### 1.3 판형 표준

| 판형 | 비율 | 주요 용도 |
| --- | --- | --- |
| 정방형 | 1:1 (1080x1080px) | 인스타그램 기본, 가장 범용적 |
| 세로 직사각형 | 4:5 (1080x1350px) | 인스타그램 피드 최적 |
| 가로 직사각형 | 16:9 (1920x1080px) | 페이스북, 블로그 |
| 스토리/릴스 | 9:16 (1080x1920px) | 인스타 스토리, 틱톡 |

### 1.4 디자인 가이드 수치

- **제목 폰트**: 48pt 이상, 굵은 고딕체
- **본문 폰트**: 25pt 이상, 가독성 높은 고딕체
- **행간**: 폰트크기 x 1.618 (예: 20pt 폰트 -&gt; 32pt 행간)
- **폰트 종류**: 1\~2개로 제한
- **여백**: 충분히 확보 (답답하지 않게)

## 2. 카드뉴스 제작 워크플로우 Best Practice

### 2.1 표준 제작 프로세스 (4단계)

```
Step 1: 기획 & 트렌드 분석
  - 주제 선정 (무엇을, 누구에게, 왜)
  - 타깃 오디언스 분석
  - 트렌드/검색어 분석

Step 2: 콘텐츠 구성
  - 스토리라인 작성 (카드별 핵심 메시지)
  - 텍스트 카피라이팅
  - 전체 카드 흐름 설계 (자연스러운 연결)

Step 3: 디자인 & 이미지
  - 톤앤매너 설정 (색상, 폰트, 스타일)
  - 레이아웃 선택/설계
  - 고품질 이미지 선정/생성
  - 각 카드 디자인 제작

Step 4: 검수 & 배포
  - 가독성 체크 (모바일 기준)
  - 브랜드 일관성 확인
  - 플랫폼별 사이즈 조정
  - 게시 및 성과 분석
```

### 2.2 AI 활용 최신 프로세스 (2025\~2026)

최근에는 AI를 활용한 4단계 프로세스가 주류:

1. **AI 트렌드 분석 & 기획**: ChatGPT 등으로 주제/키워드 분석
2. **AI 콘텐츠 생성**: 카피라이팅, 이미지 생성
3. **AI 디자인 보조**: Canva AI, 레이아웃 자동 추천
4. **자동 배포 & 분석**: 플랫폼별 자동 게시

### 2.3 Tyle.io의 7단계 체크리스트

Tyle.io는 카드뉴스 전문 제작 플랫폼으로, 29개 기본 레이아웃을 제공하며, 산돌폰트 무료 제공, 채널별 사이즈 자동 조정을 지원한다.

## 3. 카드뉴스 자동화 소프트웨어 & 기술 스택

### 3.1 기존 자동화 도구/플랫폼

| 도구 | 유형 | 특징 |
| --- | --- | --- |
| **Tyle.io** | SaaS | 카드뉴스 전문. 29개 레이아웃, 산돌폰트 무료 |
| **Canva** | SaaS | 범용 디자인. 카드뉴스 템플릿 다수. Autofill API 제공 (Enterprise) |
| **망고보드** | SaaS | 한국형 디자인 툴. 인포그래픽/카드뉴스 특화 |
| **Placid** | API/SaaS | 이미지 자동 생성 API. 인스타 카드뉴스 자동화에 활용 |
| **APITemplate.io** | API | REST API로 소셜미디어 이미지 자동 생성 |
| **CraftMyPDF** | API | 템플릿 기반 이미지/PDF 자동 생성 |

### 3.2 자동화 워크플로우 조합

| 조합 | 설명 |
| --- | --- |
| **Make.com + 구글 슬라이드** | 텍스트/이미지를 슬라이드에 자동 삽입 후 이미지 내보내기 |
| **n8n + Figma** | 블로그 원고 입력 -&gt; Figma 템플릿에 데이터 바인딩 -&gt; 이미지 자동 생성 |
| **ChatGPT + DALL-E + Canva** | AI 콘텐츠 생성 + AI 이미지 생성 + 디자인 조합 |
| **OpenClaw + n8n** | 자연어로 자동화 워크플로우 제어 |

### 3.3 프로그래밍 방식 이미지 생성 기술

| 기술 | 설명 | 장점 |
| --- | --- | --- |
| **HTML/CSS -&gt; Puppeteer 스크린샷** | HTML 템플릿을 Headless Chrome으로 렌더링 후 캡처 | 웹 기술 그대로 활용, 픽셀 퍼펙트 |
| **node-html-to-image** | Puppeteer 기반 npm 패키지. Handlebars 템플릿 지원 | 간편한 API, 템플릿 로직 지원 |
| **Sharp (Node.js)** | 이미지 합성/처리 라이브러리 | 고성능, 서버사이드 |
| **Canvas API (node-canvas)** | 프로그래밍 방식 2D 그래픽 | 세밀한 제어 가능 |
| **Figma API + 플러그인** | Figma 디자인을 API로 제어 | 디자이너 협업 용이 |
| **Canva Connect API** | Canva 리소스 프로그래밍 접근 | 풍부한 템플릿 생태계 (Enterprise 필요) |

## 4. 카드뉴스 데이터 모델 핵심 엔티티

> \[검토 필요\] WebSearch에서 카드뉴스 특화 데이터 모델 자료는 찾지 못함. 아래는 도메인 분석을 바탕으로 도출한 엔티티 모델.

### 4.1 핵심 엔티티

```
Project (프로젝트)
├── id, name, description
├── target_platform: "instagram" | "facebook" | "blog" | ...
├── aspect_ratio: "1:1" | "4:5" | "16:9" | "9:16"
├── canvas_size: { width, height }  (px 단위)
└── created_at, updated_at

Template (템플릿)
├── id, name, category
├── aspect_ratio
├── theme: { primary_color, secondary_color, bg_style }
├── font_set: { title_font, body_font }
├── layout_slots: [SlotDefinition]  // 텍스트/이미지 배치 위치
└── thumbnail_url

Slide (슬라이드/카드)
├── id, project_id, order_index
├── slide_type: "cover" | "body" | "closing"
├── template_id (nullable - 커스텀 가능)
├── background: { type: "color"|"image"|"gradient", value }
└── elements: [Element]

Element (요소)
├── id, slide_id, order_index (z-index)
├── element_type: "text" | "image" | "shape" | "icon" | "logo"
├── position: { x, y, width, height }
├── rotation: number
├── opacity: number
└── type-specific properties (아래 참조)

TextElement extends Element
├── content: string
├── font_family, font_size, font_weight
├── color, alignment, line_height
└── text_decoration, letter_spacing

ImageElement extends Element
├── src: string (URL 또는 파일 경로)
├── alt_text: string  // 접근성
├── fit: "cover" | "contain" | "fill"
├── filter: string (optional)
└── license_info: { type, source, attribution }

StyleGuide (스타일 가이드 / 톤앤매너)
├── id, project_id
├── color_palette: [{ name, hex, usage }]
├── fonts: { title: FontSpec, body: FontSpec }
├── spacing: { margin, padding, line_height_ratio }
└── brand_assets: [{ type, url }]

ExportConfig (내보내기 설정)
├── format: "png" | "jpg" | "webp" | "pdf"
├── quality: number
├── platform_presets: [PlatformPreset]
└── naming_pattern: string
```

### 4.2 엔티티 관계

```
Project 1--* Slide
Project 1--1 StyleGuide
Project 1--1 ExportConfig
Slide 1--* Element
Template 1--* Slide (optional)
```

## 5. 카드뉴스 제작 실패 사례 & 주의사항

### 5.1 디자인 안티패턴

| 안티패턴 | 설명 | 해결책 |
| --- | --- | --- |
| **저품질 이미지** | 화질 깨짐, 워터마크 포함 | 고해상도 이미지 사용, 라이선스 확인 |
| **과도한 폰트** | 3개 이상 폰트 혼용 | 1\~2개로 제한 (제목 + 본문) |
| **장식 폰트 남용** | 가독성 저하 | 본문은 고딕체, 장식은 제목에만 |
| **여백 부족** | 답답하고 읽기 불편 | 충분한 마진 확보 |
| **일관성 부재** | 슬라이드마다 다른 디자인 | 통일된 톤앤매너 유지 |
| **텍스트 과다** | 한 카드에 너무 많은 텍스트 | 카드당 핵심 메시지 1개 |

### 5.2 콘텐츠 문제

| 문제 | 설명 |
| --- | --- |
| **사실 오류** | 잘못된 정보, 출처 미확인 데이터 게시 |
| **스토리 단절** | 카드 간 흐름이 자연스럽지 않음 |
| **CTA 부재** | 마지막 장에 행동 유도 없음 |
| **무성의한 배경** | 내용과 무관한 이미지 사용 |

### 5.3 법적/윤리적 문제

| 문제 | 심각도 | 설명 |
| --- | --- | --- |
| **이미지 저작권 침해** | 높음 | 무단 이미지 사용 시 3년 이하 징역 또는 3천만원 이하 벌금 |
| **폰트 라이선스 위반** | 높음 | 상업용 사용 시 라이선스 확인 필수 |
| **접근성 미비** | 중간 | 이미지 기반이라 시각장애인 접근 불가. alt text 제공 필요 |
| **개인정보 노출** | 높음 | 인물 사진, 개인정보가 포함된 이미지 주의 |

### 5.4 기술적 실패 사례

| 사례 | 원인 | 교훈 |
| --- | --- | --- |
| 플랫폼별 사이즈 미스매치 | 하나의 사이즈만 제작 | 플랫폼별 내보내기 필수 |
| 모바일 가독성 저하 | PC 기준으로만 디자인 | 모바일 우선 디자인 (small font 주의) |
| 렌더링 품질 저하 | 압축 과다 또는 잘못된 포맷 | PNG(텍스트 많을 때), JPG(사진 중심) 구분 |
